# 🤝 貢獻與協作指南 (Contributing Guide)

歡迎來到「新竹市月子照顧服務人員職業工會 - LINE 應用與行政流程自動化系統」開源專案！
為了讓大家能輕鬆協作，並保持專案歷史整潔，請參考以下簡單的步驟來提交您的修改。

---

## 1. 協作模型：先「分叉」再修改
因為這是公開專案，我們使用 **「分叉合併模型 (Fork and Pull Model)」**。
* **為什麼**： 這樣您不需要直接在我的原始倉庫寫入，可以先在您自己的副本裡放心練習。
* **操作**： 點擊頁面右上角的 "Fork" 按鈕，建立一個您自己的倉庫副本。

## 2. 開發流程：建立「主題分支」
不要直接在 `main`（主分支）上修改，應為每個新功能建立一個獨立的「主題分支」（例如: `feature-ui-update`）。
* **為什麼**： 這樣如果您同時想修兩個 Bug，代碼才不會混在一起。
* **操作**：
  1. 將 Fork 後的倉庫 Clone 到電腦裡。
  2. 建立新分支：`git checkout -b 分支名稱`。

## 3. 關於「壓扁合併 (Squash)」：不用擔心 Commit 太多
本專案會將您的 PR **「壓扁（Squash）」** 成一個單一的提交再併入。
* **這是什麼意思**： 不管您在開發過程中提交了 10 次還是 100 次（例如：「修錯字」、「再試一次」、「終於成功了」），合併時它們都會被打包成 **一個乾淨的紀錄**。
* **新手利多**： 您可以隨時 Commit 紀錄進度，不用擔心紀錄太亂。我們會幫您把歷史變整潔。

## 4. 提交 PR 的規範
當您準備好合併時，請在 GitHub 發起 Pull Request：
* **標題要清楚**： 請用一句話描述您改了什麼（例如：`修正登入頁面的按鈕顏色`）。
* **描述內容**： 請簡單說明為什麼要這項修改。
* **等待審查**： 管理員會查看您的代碼。如果需要修改，直接在原分支推送新提交即可，PR 會自動更新。

## 5. 注意事項（新手必看）
* **保持同步**： 如果專案開發了很久，請記得在合併前同步主專案的最新代碼，以避免「合併衝突」。
* **嚴禁敏感資訊**： 絕對不要在代碼裡放 API 金鑰或密碼！一旦推送到公開倉庫，全世界都能看到。
* **單一目標**： 一個 PR 最好只處理一件事，這樣審查起來最快。
