# Obsidian 畫圖測試與範例

這是一個在 Obsidian 中使用 **Mermaid** 畫軟體架構圖的測試檔案。
請在 Obsidian 中打開此檔案，並確保切換到 **「閱讀模式 (Reading view)」** 或 **「即時預覽模式 (Live Preview)」**，您就會直接看到渲染出來的圖形。

## 一：月子公會系統架構圖 (實戰分層風格)
這個版本參考了您的 [軟體需求規格書.md](file:///c:/Users/chris/Desktop/project/union/軟體需求規格書.md) 的實際架構，並以您喜歡的**「明顯分層、間隔開」**的風格進行重新排版。

我們將系統明確區分為三個區塊：**外部雲端層**、**地端核心服務層** 與 **地端資料儲存層**，並搭配了不同的色彩框線進行實體間隔：

```mermaid
flowchart TD
    %% 定義配色與節點樣式
    classDef external fill:#F5F5F5,stroke:#9E9E9E,stroke-width:2px;
    classDef line fill:#E8F5E9,stroke:#2E7D32,stroke-width:2px;
    classDef security fill:#FFFDE7,stroke:#FBC02D,stroke-width:2px;
    classDef app fill:#E3F2FD,stroke:#1565C0,stroke-width:2px;
    classDef storage fill:#FFF3E0,stroke:#EF6C00,stroke-width:2px;

    %% ==========================================
    %% 1. 外部使用者、管理員與外部服務
    %% ==========================================
    subgraph ExternalLayer [ 🌐 使用者與外部服務 ]
        direction LR
        User(["👤 使用者"]):::external
        Admin(["👤 系統管理員"]):::external
        EmbeddingAPI["☁️ 外部 Embedding API <br/> OpenAI 與 Gemini"]:::external
        GovSite["🏛️ 政府官方登記網站"]:::external
    end
    style ExternalLayer fill:#ECEFF1,stroke:#78909C,stroke-width:2px,stroke-dasharray: 5 5

    %% ==========================================
    %% 2. LINE 應用與互動層 (大應用層)
    %% ==========================================
    subgraph LineLayer [ 💬 LINE 應用與互動層 ]
        direction TB
        LinePlatform["LINE 平台 <br/> 官方帳號與 Rich Menu"]:::line
        LineBotApp["LINE Bot 應用服務 <br/> FastAPI Webhook"]:::line
    end
    style LineLayer fill:#E8F5E9,stroke:#4CAF50,stroke-width:2px,stroke-dasharray: 5 5

    %% ==========================================
    %% 3. 地端網路安全屏障 (Security Gateway)
    %% ==========================================
    subgraph SecurityBoundary [ 🛡️ 地端網路安全邊界 ]
        direction TB
        Firewall["🧱 防火牆 <br/> 限 LINE IP 白名單"]:::security
        Nginx["🌐 Nginx 反向代理 <br/> SSL 憑證加密"]:::security
        VPN["🔒 VPN 安全網關 <br/> 如 WireGuard"]:::security
    end
    style SecurityBoundary fill:#FFFDE7,stroke:#FBC02D,stroke-width:2px,stroke-dasharray: 5 5

    %% ==========================================
    %% 4. 地端核心服務層 (隔離於內網)
    %% ==========================================
    subgraph AppLayer [ ⚙️ 地端伺服器 - 核心內網服務層 ]
        direction TB
        RAGCore["🧠 RAG 檢索生成核心"]:::app
        
        subgraph DataTools [ 資料管理與自動化工具 ]
            direction LR
            Crawler["🕷️ 自動化爬蟲程式"]:::app
            DataPipeline["🧪 Data Pipeline"]:::app
            AdminUI["🖥️ 管理員 UI <br/> Streamlit"]:::app
        end
        style DataTools fill:#E1F5FE,stroke:#0288D1,stroke-width:1px
    end
    style AppLayer fill:#E0F7FA,stroke:#00ACC1,stroke-width:2px,stroke-dasharray: 5 5

    %% ==========================================
    %% 5. 地端資料儲存層 (實體隔離)
    %% ==========================================
    subgraph StorageLayer [ 💾 地端伺服器 - 資料儲存層 ]
        direction LR
        VectorDB[("向量資料庫 <br/> Chroma / Milvus / pgvector")]:::storage
        MySQL[("MySQL 資料庫")]:::storage
    end
    style StorageLayer fill:#FFF3E0,stroke:#FF9800,stroke-width:2px,stroke-dasharray: 5 5

    %% ==========================================
    %% 連線與互動關係 (Connections)
    %% ==========================================
    
    %% LINE 外部通訊流 (經防火牆與 Nginx 反向代理)
    User <--> |LINE 訊息互動| LinePlatform
    LinePlatform --> |Step 1: HTTPS Port 443| Firewall
    Firewall --> |Step 2: 驗證通過| Nginx
    Nginx <--> |Step 3: 本地轉發 Port 8000| LineBotApp
    
    %% 管理員存取流 (必須經 VPN 隧道)
    Admin --> |Step 1: VPN 連線| VPN
    VPN --> |Step 2: 內網存取 Port 8501| AdminUI

    %% 內部系統通訊
    LineBotApp <--> |查詢與生成請求| RAGCore
    RAGCore <--> |API 向量化| EmbeddingAPI
    RAGCore <--> |相似度檢索| VectorDB
    RAGCore <--> |輔助資料查詢| MySQL
    LineBotApp -.-> |導引登記連結| GovSite

    %% 自動化與讀寫
    Crawler --> |下載外部資料檔 Excel| DataPipeline
    DataPipeline --> |解析、去重並寫入| MySQL
    AdminUI <--> |查詢與修改資料| MySQL
```

## 二：AI 客服語意檢索時序圖 (實戰 Sequence Diagram)
架構圖（範例一）適合用來表達系統的「靜態結構」，但如果想要把**「動態的執行步驟、邏輯判斷（如防幻覺閥值）」**寫得更清楚，最適合使用的是**時序圖 (Sequence Diagram)**。

下面是參考您的 [軟體需求規格書.md](file:///c:/Users/chris/Desktop/project/union/軟體需求規格書.md) 設計的真實 AI 客服問答流程：

```mermaid
sequenceDiagram
    autonumber
    actor User as 👤 使用者
    participant Line as 💬 LINE 平台
    participant Bot as ⚙️ LINE Bot 服務
    participant RAG as 🧠 RAG 核心
    participant Embed as ☁️ Embedding API
    participant VDB as 💾 向量資料庫
    participant DB as 💾 MySQL 資料庫

    User->>Line: 1. 發送提問 (例如：「如何申請登記？」)
    Line->>Bot: 2. Webhook 轉發訊息
    Bot->>RAG: 3. 請求語意檢索回答
    RAG->>Embed: 4. 將提問文字轉為高維向量
    Embed-->>RAG: 5. 回傳向量資料
    RAG->>VDB: 6. 進行餘弦相似度檢索
    VDB-->>RAG: 7. 回傳最相似問題與分數 (Score)
    
    alt 相似度 > 閥值 (Threshold)
        RAG->>DB: 8a. 查詢對應的標準答案
        DB-->>RAG: 8b. 回傳標準答案
        RAG-->>Bot: 9a. 回傳確認的答案內容
    else 相似度 <= 閥值 (低於門檻)
        RAG-->>Bot: 9b. 回傳預設導引提示 (防止 AI 幻覺)
    end

    Bot->>Line: 10. 呼叫 Messaging API 發送回覆
    Line-->>User: 11. 收到回覆訊息
```

## 範例三：資料庫與爬蟲自動化時序圖 (ETL Sequence Diagram)
這個範例展示了「資料庫與自動化」部分的執行步驟。它描繪了定時排程器觸發爬蟲後，爬蟲、**Data Pipeline** 以及 **MySQL 資料庫** 之間的互動流程（包含檔案解析、迴圈處理與 Insert/Update 去重邏輯）：

```mermaid
sequenceDiagram
    autonumber
    participant Scheduler as ⏰ 定時排程器 (Cron)
    participant Crawler as 🕷️ 自動化爬蟲程式
    participant Gov as 🏛️ 政府登記網站
    participant Pipeline as 🧪 Data Pipeline (ETL)
    participant DB as 💾 MySQL 資料庫

    Scheduler->>Crawler: 1. 觸發定時爬取任務 (例如：每日凌晨)
    Crawler->>Gov: 2. 模擬登入並檢查是否有新資料
    Gov-->>Crawler: 3. 回傳資料清單或檔案連結
    
    opt 發現有新資料或新檔案
        Crawler->>Gov: 4. 下載最新檔案 (Excel/PDF)
        Gov-->>Crawler: 5. 回傳檔案內容
        Crawler->>Pipeline: 6. 將檔案交給 Pipeline 進行解析
        
        loop 逐筆解析與去重比對
            Pipeline->>DB: 7. 依唯一識別碼 (如身分證字號) 查詢資料庫
            DB-->>Pipeline: 8. 回傳此筆資料是否存在
            
            alt 資料庫中「不存在」此筆資料
                Pipeline->>DB: 9a. 執行 INSERT 寫入全新資料
            else 資料已存在但內容有變更
                Pipeline->>DB: 9b. 執行 UPDATE 更新該筆資料
            end
        end
        
        Pipeline-->>Crawler: 10. 回報資料同步完成
    end
```

## 範例四：系統結構目錄樹 (Mermaid Mindmap 風格)
這個範例展示了如何使用 `mindmap` 語法，以視覺化的「心智圖/樹狀圖」來呈現您專案的系統功能層級分類。

這比一般的流程圖更適合用來展示「功能大綱」或「系統分類」：

```mermaid
mindmap
  root((月子公會系統))
    💬 LINE 應用與互動層
      LINE 官方帳號
      LINE Rich Menu
      FastAPI Webhook 服務
    ⚙️ 地端核心服務層
      RAG 檢索生成核心
      Data Pipeline
      自動化爬蟲程式
      Streamlit 管理 UI
    💾 地端資料儲存層
      MySQL 關聯式資料庫
      Chroma 向量資料庫
    🌐 外部與雲端平台
      LINE Messaging API
      外部 Embedding API
      政府官方登記網站
    🛡️ 網路與資訊安全防護
      地端防火牆 (LINE IP 白名單)
      Nginx 反向代理 (SSL 加密)
      WireGuard VPN 安全隧道
      資料庫安全隔離 (監聽本地)
      環境變數金鑰隔離 (.env)
```

## 如何修改這些圖？
1. 在 Obsidian 中切換到 **「原始碼模式 (Source mode)」** 或直接雙擊上面的圖表。
2. 您會看到由 ` ```mermaid ` 和 ` ``` ` 包起來的文字。
3. 您可以直接修改裡面的文字（例如把 `PostgreSQL` 改成 `MySQL`），圖表就會即時更新！
