# 🧪 帳務假數據生成與關聯測試規格 (SSOT)

本文件定義財務與帳務模組開發期間，**測試假數據 (Fake Data) 的生成規則與跨表關聯 (JOIN) 原則**。
所有帳務假資料生成腳本（如 `scripts/generate_fake_finance.py`）與匯入對帳腳本均必須以此文檔為開發對照依據，以防範數據結構脫鉤。

---

## 1. 跨表關聯 (JOIN) 設計原則

為了在沒有實體外鍵約束（或尚未建立關聯）時驗證對帳邏輯，假數據必須使用「同一批案件編號（查詢序號）」串聯：

```
+------------------+         +------------------+         +------------------+
|     clients      |         |      orders      |         |     payments     |
+------------------+         +------------------+         +------------------+
| 主鍵: 案件編號    | <====== | 主鍵: 案件編號    | <====== | 關聯鍵: 案件編號  |
| (查詢序號 9碼)   |         | (查詢序號 9碼)   |         |                  |
+------------------+         +------------------+         +------------------+
         ^                                                         ^
         |                    解析對照還原後 6 碼                   |
         +---------------------------------------------------------+
                                 虛擬帳號
                            (99781699 + 115001)
```

1. **基礎案件編號**：假數據統一使用 **「查詢序號（案件編號）」** 作為貫穿核心（如 `115000001` 至 `115000010`）。
2. **虛擬帳號對照公式**：
   * 虛擬帳號後 6 碼 $= \text{年度(3碼)} + \text{流水號後3碼(3碼)}$。
   * 範例：案件編號 `115000001` $\rightarrow$ 縮寫為 `115001` $\rightarrow$ 完整虛擬帳號為 `99781699115001`。

## 2. 測試金額固定數值規格

> [!IMPORTANT]
> **重要備註 (ponytail)**
> 目前帳務系統與動態天數計價公式尚未建立。為了專注於驗證 Pipeline 對帳邏輯與狀態機移轉，現階段**所有假數據的財務金額均採用「固定數值」進行測試**。

| 財務欄位 | 模擬測試固定金額 (TWD) | 說明 |
| :--- | :--- | :--- |
| **應收總額 (`amount_receivable`)** | **`$80,000`** | 案件合約總金額 |
| **已收訂金 (`deposit_received`)** | **`$12,000`** | 簽約時客戶應付之訂金 |
| **已收尾款 (`balance_received`)** | **`$68,000`** | 生產服務結束後客戶應付之尾款 |
| **轉帳月嫂 (`caregiver_fee`)** | **`$60,000`** | 工會撥款給服務人員/月嫂的費用 |

## 3. 測試用流水帳 (帳務.xlsx) 混雜與驗證原則

在產生的 `帳務.xlsx` 的 `合作社帳戶` (流水帳) 中，必須混雜以下數據以驗證對帳腳本的**過濾與防錯能力**：

1. **正常月子服務進帳**：
   * 包含存入 $12000$ (備註虛擬帳號如 `99781699115001`)。
   * 包含存入 $68000$ (備註虛擬帳號如 `99781699115001`)。
2. **正常月子服務支出**：
   * 包含轉帳支出 $60000$ 給月嫂的明細。
3. **應被忽略的雜訊**：
   * 故意混入課程虛擬帳號的存入紀錄 (如 `99781611302045`)，對帳腳本讀取時**必須忽略**此筆交易，不可寫入 `payments` 表。
   * 故意混入會員費虛擬帳號的存入紀錄 (如 `99781600004567`)，對帳腳本讀取時**必須忽略**。
